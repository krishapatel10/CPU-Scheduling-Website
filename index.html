CPU SCHEDULING



CPU scheduling is done by switching the CPU processes.
By CPU scheduling, the operating system can make the computer more predictive.
CPU scheduling is also known as 'Process scheduling'.




Why CPU Scheduling?

In Multiprogramming systems, the Operating system schedules the process on the CPU to have the maximum utilization of it and this procedure is called CPU scheduling.


The task of Operating system is to optimize the utilization of resources. If most of the running processes change their state from running to waiting then there may always be a possibility of deadlock in the system. Hence to reduce this overhead, the OS needs to schedule the jobs to get the optimal utilization of CPU and to avoid possibility of deadlock.


Objectives of CPU scheduling algorithms:
1. Maximize Thoroughput
2. Minimize response time
3. Minimize turnaround time
4. Minimize waiting time
5. Maximize CPU efficiency
6. Ensure fairness for all jobs

Terminologies


Some of the terminologies used in the algorithm are as follows:



  • Burst Time

    The time required by the process to complete execution. It is also called running time.

  • Turnaround Time

    Turn around time is given by the difference in completion time and arrival time.

  • Arrival Time

    The time when a process enters in a ready state is called the arrival time.

  • Waiting Time

    Waiting time is the difference in turnaround time and burst time.

  • Completition Time

    The time when process gets completed and exits from the system is called its completion time.

First Come First Serve

FCFS provides an efficient, simple and error-free process scheduling algorithm that saves valuable CPU resources. It uses nonpreemptive scheduling in which a process is automatically queued and processing occurs according to an incoming request or process order. FCFS derives its concept from real-life customer service.

Shortest Remaining Job First

The Shortest Remaining Job First (SRJF) is the preemptive version of SJF scheduling. In this scheduling algorithm, the process with the smallest amount of time remaining until completion is selected to execute. Processes having same arrival time will convert SRTF to SJF.

Round Robin Scheduling

Round Robin is the preemptive process scheduling algorithm. Each process is provided a fix time to execute, it is called a quantum. Once a process is executed for a given time period, it is preempted and other process executes for a given time period. Context switching is used to save states of preempted processes.

Shortest Job First

The Shortest Job First (SJF) is a scheduling policy that selects the waiting process with the smallest execution time to execute next. It is also known as Shortest Job Next (SJN) or Shortest Process Next (SPN). It is a non-preemptive scheduling algorithm.
